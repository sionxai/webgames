# 프로젝트 에이전트 지침 (AGENTS.md)

이 문서는 `webgames` 프로젝트 및 하위 디렉토리 트리 전체에 적용되는 에이전트(Antigravity 등) 작동 지침입니다.

---

## 🎯 범위와 우선순위
- 본 지침은 이 파일이 위치한 디렉터리 트리 전체에 적용됩니다.
- 더 깊은 경로에 `AGENTS.md`가 존재할 경우 해당 파일의 지침이 우선합니다.
- 지침 간 우선순위: `system` > `developer` > `user` > `AGENTS.md`

---

## 🗣️ 기본 언어 · 톤
- **기본 언어**: 한국어를 기본 커뮤니케이션 언어로 사용합니다. 코드, 터미널 명령, 함수/변수 식별자는 영문 그대로 유지합니다.
- **톤 & 매너**: 간결하고 친절하게, 불필요한 장황함 없이 **결과 중심**으로 설명합니다.

---

## 📋 수정 전 승인 절차 (중간 이상 작업에 적용)

다음 조건에 해당하는 작업은 변경 전 사용자 승인 절차를 거칩니다:
- 파일 수정 / 추가 / 삭제가 필요한 경우
- 작업 범위가 1개 파일 10줄 이상이거나, 영향 범위가 넓은 경우
- 타입 / 데이터 구조 변경, API / DB 스키마 변경

### 승인 절차 3단계:
1. **3줄 요약 (What / Why / How)**
   - 무엇을, 왜, 어떻게 바꿀지 한 줄씩 총 3줄로 요약합니다.
2. **구체적 수정 계획 (체크리스트)**
   - 실행 가능한 단계 목록(5~8개 내외)으로 제시합니다.
   - 파일 경로, 비즈니스 영향, 타입 변경 등 핵심을 포함합니다.
   - 단계별 진행 상태는 `implementation_plan.md` 계획 문서로 공유하고 업데이트합니다.
3. **확인 요청 (명시적 동의)**
   - “위 계획으로 진행할까요? (예/아니오, 수정사항 제안 가능)” 형태로 명시적인 동의를 구합니다.
   - 사용자 동의 없이 파일 변경 도구를 실행하지 않습니다.

> **예외**: 매우 단순한 수정(오타, 문구 변경 등)은 요약만 제공하고 승인 절차를 생략할 수 있습니다.

## 🛠️ 커뮤니케이션 & 도구 사용

- **사전 안내**: 무거운 작업이나 터미널 명령 실행 전, 1–2문장으로 즉시 다음 행동을 안내합니다.
- **검색 도구**: 파일 및 텍스트 검색 시 Antigravity의 `grep_search` 도구 또는 `ripgrep (rg)`을 우선 사용합니다.
- **파일 편집**: 
  - 코드 편집 시 `replace_file_content` / `multi_replace_file_content` 도구를 우선 사용합니다.
  - 대량 치환, 자동 생성 결과물 작성, 다중 파일 신규 생성 시에는 `write_to_file` 또는 스크립트 도구를 사용합니다.
- **파일 조회**: 큰 파일 조회 시 `view_file` 도구의 `StartLine` / `EndLine`을 지정하여 250줄 내외 단위로 읽고 필요한 구간을 탐색합니다.
- **무거운 명령**: 빌드, 린트, 테스트 등 시간이 오래 걸리는 작업은 사전 안내 후 진행합니다.

## 💻 코드 변경 원칙

- **최소 변경**: 스코프를 불필요하게 넓히지 않고, 문제의 근본 원인을 최소 변경으로 해결합니다.
- **스타일 유지**: 기존 프로젝트의 코드 스타일 및 포맷팅 규격을 엄격히 준수합니다.
- **불필요한 수정 금지**: 관련 없는 리팩터링, 대규모 포맷팅, 불필요한 외장 패키지 추가를 금지합니다.
- **TypeScript 규격**: `null`과 `undefined`의 구분을 명확히 하며, 선언 타입과 반환 타입을 일치시킵니다.

## ⚡ Next.js / ESLint 가이드

- **ESLint**: ESLint 경고는 실제 기능 저해 요인이 있는 경우에만 수정합니다. 단, **타입 오류(Type Errors)는 반드시 해결**해야 합니다.
- **이미지 최적화**: `<img>` 태그를 `next/image`로 치환하는 작업은 성능 및 품질상의 이득이 분명할 때만 실시합니다.

## 🔒 보안 & 환경 변수

- **비밀키 보관**: Secret Key, API Key 하드코딩을 절대 금지합니다. `.env.local`을 사용하고 Git에 커밋되지 않도록 합니다.
- **키 분리**: `OPENAI_API_KEY`, `GEMINI_API_KEY`, Firebase 설정 값 등은 서버 side / 클라이언트 side 구분을 엄격히 준수합니다.
- **로그 민감정보 마스킹**: 콘솔 및 로그에 민감정보 전체를 출력하지 않으며, 필요 시 앞 6~10자만 노출하고 나머지는 마스킹 처리합니다.

## 🎨 스토리보드 모듈 (프로젝트 특화)

- `/api/storyboard` 응답은 `{ ok: boolean }`을 기준으로 분기하며, 실패 시에는 사유(`reason`)를 명확히 제공합니다.
- 스타일 조회 실패 시 내장 Fallback 시스템을 사용하며, Fallback이 없을 경우에는 명확한 에러를 반환합니다.
- 프롬프트 구성 시 `natural` / `json` 모드를 정확히 분기하여 적용합니다.

## 🛑 금지 사항

- 사용자 요청 없는 대규모 리네이밍, 자동 포맷팅, 폴더 구조 이동.
