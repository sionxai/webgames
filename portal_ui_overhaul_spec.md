# UI 대개편 P1 — 가챠 화면 재건 (게임 로직 무수정)

## 0. 왜 이 방식인가
가챠는 `app.js` 497KB·`index.html` 82KB(인라인 style 145개)로, 구조를 직접 뜯으면 게임이 깨진다.
따라서 **기존 파일을 고치지 않고, 뒤에 로드되는 CSS와 DOM 정리 스크립트로 덮는다.** 문제가 생기면 두 파일만 빼면 원복된다.

## 1. 목표
모바일에서 읽을 수 없는 현재 화면을 정리한다. 현재 확인된 결함:
- 퀘스트 패널·월드 채팅이 본문 위에 반투명하게 겹쳐 글자가 서로 뚫려 보임
- 자원 표시가 3줄로 흩어짐(포인트/골드/다이아/펫뽑기권/성수)
- 상단 버튼 6개(사용자 페이지·관리자 페이지·로그아웃·퀘스트·옵션 등)가 무질서하게 줄바꿈
- 탭이 2단(뽑기/확률/통계/결과로그 + 장비/펫/캐릭터)
- 포털 복귀 버튼이 게임 타이틀을 가림

## 2. 변경 파일 (목록 밖 수정 금지)
- 신규: `games/gacha/hanpan-ui.css` — 개선 스타일 전부
- 신규: `games/gacha/hanpan-ui.js` — DOM 재배치·토글만 (게임 상태·로직 접근 금지)
- 수정: `games/gacha/index.html`, `login.html`, `signup.html` — **위 두 파일을 로드하는 태그 추가만**(각 1~2줄). 다른 마크업 수정 금지
- 절대 금지: `app.js`, `battle.js`, `pvp.js`, `styles.css`, `constants.js`, `firebase.js`, 기타 게임 스크립트

## 3. 계약

### 3.1 레이어 정리 (최우선)
- 퀘스트·옵션·상점·우편함 등 패널은 **한 번에 하나만** 열리고, 열리면 전체 백드롭(dim + blur)을 깔고 본문 스크롤을 잠근다.
- 월드 채팅은 기본 접힘. 우하단 플로팅 버튼(말풍선 아이콘)으로 열고 닫는다. 열리면 하단 시트로 올라오고 백드롭을 깐다.
- z-index 층: 본문 0 → 하단 탭바 100 → 백드롭 900 → 패널 1000 → 포털 복귀 버튼 2147483000(기존 유지).
- 패널에는 닫기(×) 버튼과 Esc·백드롭 클릭 닫기를 보장한다. 기존 닫기 버튼이 있으면 그것을 재사용한다.

### 3.2 상단 헤더 압축
- 자원(포인트·골드·다이아·펫뽑기권·성수)은 **아이콘 + 숫자 칩** 한 줄로 압축하고, 넘치면 가로 스크롤(스크롤바 숨김).
- 라벨 텍스트("포인트:", "골드:")는 아이콘/툴팁으로 대체해 폭을 줄인다.
- 상단 버튼들은 **오른쪽 끝 아이콘 버튼 2개**로 축약: ① 프로필(닉네임·로그아웃·사용자 페이지·관리자 페이지) ② 메뉴(퀘스트·옵션·기타). 나머지는 그 안의 목록으로 옮긴다.
- 왼쪽 상단은 포털 복귀 버튼 자리이므로 `padding-left: 52px` 이상 확보.
- 게임 부제("확률 직접 지정 · 1/10/100회 …")와 버전 표기는 모바일에서 숨긴다.

### 3.3 탭 구조
- 2단 탭을 1단으로 정리한다: 주 탭(뽑기/확률/통계/기록)은 세그먼트 컨트롤로, 하위 선택(장비/펫/캐릭터)은 주 탭 아래 칩 행으로.
- 하단 탭바(뽑기/장비/전투/상점/PVP)는 유지하되 높이·아이콘·활성 표시를 정리하고 `env(safe-area-inset-bottom)`을 반영한다.

### 3.4 시각 언어
- 한판 포털 톤과 이어지도록 다음 토큰을 `hanpan-ui.css` 상단에 정의하고 사용한다:
  `--hp-bg:#0f1117; --hp-surface:#171a22; --hp-surface-2:#1f2430; --hp-border:#2b3140; --hp-text:#e8ecf4; --hp-muted:#98a1b3; --hp-accent:#e9a44b; --hp-accent-2:#ff8a3d; --hp-radius:14px;`
- 카드·패널·버튼의 반경/그림자/여백을 통일한다. 터치 타겟 최소 44×44.
- 기존 게임의 파란 강조색은 유지해도 되나, 버튼 위계를 만든다(주요 1개만 채움 버튼, 나머지는 외곽선/고스트).

### 3.5 hanpan-ui.js 제약
- 하는 일: 헤더 요소를 새 컨테이너로 **이동**(appendChild), 패널 열림/닫힘 클래스 토글, 채팅 접기 버튼 생성, 백드롭 요소 생성, 스크롤 잠금.
- 금지: 게임 상태 읽기/쓰기, 기존 이벤트 리스너 제거, 요소 삭제, `app.js`가 참조하는 **id·class 변경**(이동만 허용, 속성 보존).
- 대상 요소가 없으면 조용히 건너뛴다(다른 페이지·업데이트로 구조가 바뀌어도 깨지지 않게).
- `DOMContentLoaded` 이후 실행하고, 게임이 나중에 렌더하는 요소는 `MutationObserver`로 한 번만 처리한다.

## 4. 수용 기준 (Claude가 브라우저로 검증)
- 390×844에서: 헤더 1줄, 자원 칩 한 줄, 상단 버튼 2개, 겹쳐 보이는 패널 0건.
- 퀘스트를 열면 본문이 가려지고 배경 글자가 비쳐 보이지 않는다. Esc·백드롭·× 로 닫힌다.
- 채팅은 기본 접힘, 플로팅 버튼으로 열림.
- 뽑기·전투·상점·PVP 이동과 실제 뽑기 실행이 이전과 동일하게 작동한다(게임 로직 회귀 0).
- 콘솔 오류 0. `npm run build` exit 0.
