# UI 대개편 P2-B — forge 심플 모드 (강화 핵심만, 나머지는 토글)

## 1. 목표
대장간 화면의 정보 밀도를 낮춘다. 기본은 **간단히 모드**: 강화에 필요한 핵심만 보이고, 나머지는 "자세히" 토글로 켠다. 게임 규칙·수치·판정은 그대로다(표시만 바뀜).

## 2. 변경 파일 (목록 밖 수정 금지)
- 수정: `src/components/game/EnhancePanel.tsx`, `src/App.tsx`(토글 상태·전달 필요분만), `src/index.css`
- 필요시 신규: `src/components/game/panelDensity.ts`(localStorage 헬퍼 소형)
- 절대 금지: `src/services/**`, `src/constants/**`, `src/types/**`, `src/components/game/CombatCanvas.tsx`, AI 관전·기록·도감 관련 파일, `window.webgamesAgent` 브리지 경로, waitdog·home·games/**

## 3. 계약

### 3.1 토글
- EnhancePanel 헤더(“강화 설계대” 행) 우측에 세그먼트 `간단히 | 자세히`.
- 기본값 간단히. `localStorage['forge_ui_density_v1']`에 `'simple' | 'detail'` 저장, 새로고침 후 유지.
- 저장 실패(사파리 프라이빗 등)는 무시하고 세션 내 상태만 유지.

### 3.2 간단히 모드에 남기는 것 (이 외 전부 숨김)
1. 현재 검 카드: 스프라이트·이름·+단계·공격력 → 성공 시 이름·공격력 (기존 비교 카드 유지하되 여백 압축)
2. **성공 확률 하나의 큰 숫자** + "초반 보호 적용" 뱃지(해당 시)
3. 강화 비용 + 보유 골드 (한 줄)
4. `+1 강화` CTA (기존 그대로)
5. 매각 요약 한 줄: "지금 매각하면 X G · 예상 정수 Y" + 매각 버튼 (기존 매각 카드 압축판)
6. 균열 칩: `균열 0/3` 소형 칩 (3.4 예외 참조)
7. **차단·경고는 숨기지 않는다**: 강화 버튼이 비활성인 이유(충전 부족·파괴 상태·골드 부족 등), 파괴/복구 UI, 수리 필요 상태 등 행동에 필요한 안내는 간단히 모드에서도 항상 표시. 기존 문구 재사용, 압축 표시는 허용.

### 3.3 간단히 모드에서 숨기는 것
- 새 검 계열 선택(설계도) 카드 — 단 +0 검이고 해금 계열이 2개 이상이면 "계열 변경 ▸" 한 줄 버튼으로 접어서 노출(누르면 해당 카드만 펼침)
- 4분포 상세(단계 유지·균열 발생·단계 하락 %), 연속 실패 보정
- 촉매·심연/제련 충전 상세 카드(잔여 충전 수는 차단 안내 문구에 포함되면 충분)
- 초월·유물·조각 상세, 기록성 정보
- 기타 보조 설명 텍스트

### 3.4 상태 우선 규칙
파괴 대기·균열 3 임박(2/3 이상)·수리 필요 등 위험 상태에서는 해당 상세 블록을 간단히 모드에서도 자동 노출한다(안전 정보 우선). 위험이 해소되면 다시 접힌다.

### 3.5 자세히 모드
기존 화면 그대로(요소 삭제 금지). 토글만 왕복해도 상태·입력이 유실되지 않아야 한다.

### 3.6 구현 방식
- 조건부 렌더가 아니라 **섹션 래퍼에 클래스 부여 + CSS 숨김**(`display:none`)을 기본으로 하되, 컴포넌트 구조상 조건부 렌더가 더 깔끔한 곳은 허용. 단 게임 상태 계산·핸들러는 항상 실행 구조 유지(로직 분기 금지).
- 새 클래스는 `forge-density-*` 접두어. 기존 클래스·문구·aria 속성 보존.
- 모바일(≤480px)과 가로형(≥900px 2열) 모두에서 성립해야 한다.
