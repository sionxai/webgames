## P9 Fixed Decisions

- +10→+11부터 +19→+20까지 현재 단계마다 전용 보스와 전용 촉매 재료가 하나씩 대응한다.
- 보스는 반복 사냥할 수 있다. 골드·정수 최초 처치 보상은 기존처럼 런당 한 번만 지급하지만 재료 드롭은 매 처치마다 독립 판정한다.
- 드롭률/천장은 +10~+19 순서로 `60/55/50/45/40/35/28/20/15/10%`, `3/3/4/4/5/5/6/8/10/15회`를 사용한다. 자연 드롭이 없으면 표기된 번째 처치에서 확정 지급하고, 재료를 획득하면 해당 보스의 천장 카운터를 0으로 초기화한다.
- 재료 한 개는 +10~+14에서 3회, +15~+17에서 2회, +18~+19에서 1회의 강화 충전을 제공한다. 해당 단계 강화 시 충전 1을 먼저 소모한다.
- +10 첫 재료는 시스템 학습을 위해 최초 발견 시 확정 지급한다. 이후에는 공개된 드롭률과 천장을 적용한다.
- 미장착 재료와 천장 진행도는 계정에 영구 보존한다. 검에 활성화된 충전은 성공 시 해당 단계에서 제거되고, 검 파괴·매각·런 정산 시 전부 제거한다.
- 기존 저장 키를 유지하고 신규 inventory/pity/charge 필드는 안전한 기본값으로 마이그레이션한다.
- 생성 이미지는 로컬 WebP 아틀라스로 저장하며 보스/재료 이름·확률·셀 위치는 하나의 데이터 계약에서 관리한다.

## P13-P18 Fixed Decisions

- 조우는 +5에서 열린다. 일반 적 처치마다 저장된 셔플백 카드를 한 장씩 공개하고, 보스 카드가 나오면 처치 전까지 같은 보스가 유지된다.
- 셔플백 크기는 `+5~9: 6`, `+10~13: 8`, `+14~17: 12`, `+18: 20`, `+19: 30`이다. 카드 위치는 최초 +5 도달 및 직전 보스 해결 직후에만 한 번 추첨하고 프로필에 저장한다. 단계 변화와 새로고침은 재추첨 사유가 아니다.
- 보스 외형은 조우가 공개된 순간의 강화 단계에 고정한다. +5~+9는 신규 중간보스 5종, +10~+19는 기존 단계별 보스 10종을 사용한다.
- 진행 촉매 드롭에는 RNG를 쓰지 않는다. `제련의 불씨`는 +10~+13 강화에, `심연의 인장`은 +14~+19 강화에 시도당 1회 소비한다.
- +5~+9 보스는 제련의 불씨를 최대 4회분까지 채운다. +10~+13 보스는 현재 막힘 방지를 위해 제련의 불씨를 3회분까지 채우고 심연의 인장을 최대 2회분 추가한다. +14~+17 보스는 심연의 인장을 2회분, +18~+19 보스는 1회분까지 채운다. 전체 보유 상한은 제련 4회, 심연 3회다.
- 현재 구간에서 실제 소비 가능한 진행 충전이 있으면 셔플백 진행을 멈춘다. 다른 구간용 충전, 희귀 조각, 레거시 재료는 정지 조건이 아니다. 충전이 0이 되면 기존 카드 위치에서 재개한다.
- 진행 촉매와 셔플백은 `weaponId`에 귀속한다. 성공·유지·균열·하락·파괴 대기·모의 광고 복구에서는 보존하고, 매각·런 정산·복구하지 않은 파괴 정산으로 새 검을 만들 때 초기화한다.
- +18 보스의 같은 무기 최초 적격 처치는 `3%`, +19는 `1%`로 완제품 초월 유물을 판정한다. 실패하면 해당 조각 1개를 확정 지급하며 40/100개에서 자동 합성한다. 희귀 판정은 같은 `weaponId + 단계`에서 한 번뿐이다.
- 같은 검의 첫 +20 강화 시도는 종말 조각 1개를, +20 성공은 종말 조각 2개를 추가 지급한다. 광고는 유물 확률·조각·진행 촉매에 영향을 주지 않는다.
- 최종 강화 분포는 성공/유지/균열/하락을 한 순수 계산에서 만든다. 균열 저항으로 감소한 실패 질량은 유지에 귀속하고, 실제 판정은 이 누적 분포에 단일 난수 한 번만 사용한다. 네 값의 내부 합계는 정확히 100이다.
- 기존 `catalystInventory`, `catalystPity`, `discoveredCatalysts`, `activeCatalystCharges`는 신규 진행 재료로 환전하지 않고 읽기 전용 레거시 아카이브로 보존한다.
- 실제 광고 SDK, 외부 서버 랭킹, 다중 탭 부정 방지와 서버 권위 판정은 이번 프로토타입 범위에서 제외한다.

## P19-P24 Fixed Decisions

- 사람 플레이와 AI 관전 플레이는 서로 다른 로컬 저장소를 사용한다. AI 실행은 사람의 골드·정수·검·기록을 변경하지 않는다.
- 골드 매각은 골드만 지급하고 현재 런을 이어 간다. 자발적 정수 추출은 +5 이상에서 `floor(level^1.5 × 0.7)` 정수를 지급하며 새 런을 시작한다. 파괴 정산은 같은 기준의 25%만 회수한다.
- 수리 횟수는 현재 검에 기록한다. 수리비는 강화비의 `0.5 + 0.5 × 기존 수리 횟수` 배에서 시작해 수리 기술 할인을 적용하며, 수리 또는 광고 복구를 사용한 기록은 순수 기록이 아니다.
- 실패 보정은 목표 단계별로 저장한다. 예를 들어 +20 도전 실패 후 +18로 하락했다가 +19에 복귀해도 +20 보정은 유지되며, 해당 목표에 성공하거나 새 검을 만들 때만 초기화된다.
- 최고 기록은 검 ID, 계열, 최고 단계, 해당 검의 강화 횟수, 수리 횟수, 광고 복구 횟수, 인간/에이전트 구분, 달성 시각을 스냅샷으로 저장한다. 현재 검의 이후 변화나 매각은 기록을 변경하지 않는다.
- +20 성공 시 종말의 마왕 조우를 즉시 활성화하고, 한 번 처치한 뒤에는 반복 셔플백을 만들지 않는다.
- 고대 설계도로 해금된 검 계열은 +0 새 검에서만 선택할 수 있다.
- AI 브리지는 `observe`, `actions`, `act`, `exportTrace`를 제공한다. 관찰에는 보스 셔플백의 숨은 위치를 노출하지 않고, 행동은 허용 목록과 기존 도메인 검증을 통과해야 한다.
- 데모 에이전트는 공격형·안전형·혼합형 전략, 공개용 한 줄 판단, 시작·일시정지·배속·중지와 최근 행동 타임라인을 제공한다. 공개 문구는 내부 추론이 아니라 관전용 전략 요약이다.
- AI 관전 화면은 읽기 전용이다. 자동 사냥은 데모 런이 실행 중일 때만 진행하며 관전자의 포인터·키보드·강화·정산·영구 성장 입력은 AI 저장을 바꾸지 않는다. 외부 에이전트는 공개 브리지 행동으로만 상태를 변경한다.
- v3 마이그레이션 원문은 사람/AI별 `before_schema_v3` 키에 보존하고 기존 v2 백업은 덮어쓰지 않는다. 이미 v3 백업이 있으면 main 데이터를 자동 교체하지 않고 fail-closed한다.
- 로컬 AI 기록은 사람 기록과 분리하고 공식 서버 기록이 아님을 표시한다. 외부 계정, 외부 AI 호출 비용, 서버 권위 랭킹, 다중 탭 부정 방지는 이번 MVP 범위에서 제외한다.

- 포털·Forge 및 다른 게임 파일은 수정하지 않는다. 새 외부 의존성과 새 이미지 생성은 이번 범위에 포함하지 않는다.

## Fixed Decisions
